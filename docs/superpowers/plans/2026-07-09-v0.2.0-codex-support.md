# v0.2.0 Codex Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Release passdown v0.2.0 as a safe, host-aware plugin with first-class Claude Code and Codex distribution.

**Architecture:** Keep the three portable skills as the shared workflow core. Add host-specific executor guidance and native manifests at the distribution edge, while installer and CI scripts enforce deterministic copies, version agreement, and release integrity.

**Tech Stack:** Markdown Agent Skills, Bash, JSON plugin manifests, GitHub Actions, OpenSpec CLI.

## Global Constraints

- Preserve the existing `v0.1.0` tag; never move or recreate it.
- Record creation of the missing `v0.1.0` GitHub prerelease as pending external work.
- Use short-lived branches and pull requests; never push implementation directly to `main`.
- Do not maintain long-lived `0.2.x` release branches while the project is in v0.
- Keep Claude Code and Codex manifests on the same SemVer version.
- Do not delete or overwrite pre-existing user changes after a failed dispatch.
