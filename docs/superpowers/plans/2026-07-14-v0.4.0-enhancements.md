# v0.4.0 Enhancement Drop Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Per the passdown convention, run `passdown-dispatch` before executing.

**Goal:** Close the handover loop (pickup skill + machine-readable handoff logs), make dispatch decisions durable in plan files, detect install-channel violations, and guard the CHANGELOG link-reference format.

**Architecture:** Four independent enhancements land as separate commits on one short-lived branch off `release/v0.4.0`, each following the repo's doc-contract TDD pattern (extend `tests/skills.sh` / `tests/docs.sh` / new `tests/doctor.sh` first, watch it fail, then edit the SKILL.md/script/doc until green). The drop closes with a `0.4.0-beta.2` version bump so beta testers receive it via `plugin marketplace update`.

**Tech Stack:** Bash (bash-3.2-compatible, shellcheck-clean), markdown skills, GitHub Actions.

## Global Constraints

- Branch: `feat/v0.4.0-enhancements` off `release/v0.4.0`; PR targets `release/v0.4.0`, rebase-merge, linear history.
- Conventional Commits, English, imperative; **no AI attribution trailers** (repo AGENTS.md policy overrides any tool default).
- Every shell change passes `bash -n` + `shellcheck`; full gate (`tests/*.sh`, `scripts/validate-plugin.sh`, `scripts/check-version.sh`) green before push.
- Skills stay workspace-agnostic: no workspace facts inside `plugins/passdown/skills/`.
- `VERSION` and all three manifest versions move in lockstep (`scripts/check-version.sh`).

## Dispatch routing (recorded 2026-07-14)

All tasks routed `[dispatch: main]`: skill-prose changes are judgment-heavy prompt work; the doctor script and test edits are interleaved with those conventions and share one branch/gate run. No native subagent authorization was given; external executors add no benefit at this size.

```bash

pass "every CHANGELOG version heading has a link reference"
```
