# passdown OpenSpec schema

Custom OpenSpec workflow schema. Not committed as a full `schema.yaml` yet —
it is generated by forking the built-in `spec-driven` schema so it always
matches the installed OpenSpec version:

```bash
openspec schema fork spec-driven passdown
# edit ~/.local/share/openspec/schemas/passdown/schema.yaml, then copy it here
```

Customizations to apply on top of `spec-driven`:

1. **Self-contained tasks** — append to the `tasks` artifact instruction:
   every task in `tasks.md` must carry enough context (file paths, done
   criteria) that an external executor can complete it without reading any
   conversation history.
2. **Task ID conventions** — if the workspace uses task IDs (e.g.
   `PKG-0005`), the change name and commit scopes must embed them.
3. Keep everything else from `spec-driven` unchanged.
